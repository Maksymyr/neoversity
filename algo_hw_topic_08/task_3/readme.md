# Завдання 3 — об'єднання кабелів з мінімальною вартістю

Дано список довжин кабелів. На кожному кроці беремо два кабелі, з'єднуємо їх (вартість = сума їх довжин, новий кабель повертається у пул). Треба знайти такий порядок об'єднань, який мінімізує сумарну вартість усіх з'єднань.

## Запуск

```bash
python cable_merge.py        # демо на [4, 3, 2, 6] → 29
python test_cable_merge.py   # тести
```

## Реалізація

Жадібний алгоритм на **min-heap** (`heapq`):

1. Покласти всі довжини в min-heap.
2. Поки в heap'і ≥ 2 елементів:
   - вийняти два найменших;
   - додати їх суму до загальної вартості;
   - покласти суму назад у heap.
3. Повернути загальну вартість.

Складність — `O(n log n)`: `n−1` ітерацій, кожна — два `heappop` + один `heappush`, кожна операція `O(log n)`.

## Чому жадібний вибір правильний

Завдання еквівалентне побудові **дерева Хаффмана**: кожне з'єднання — внутрішній вузол з двома дітьми, його «вартість» = сума ваг дітей, а загальна вартість = сума ваг усіх внутрішніх вузлів. Кожен «листок» (вихідний кабель довжиною `Lᵢ`) увійде у фінальну суму рівно стільки разів, скільки разів він був на шляху до кореня — тобто його глибина в дереві.

Хаффмен довів, що жадібний вибір **двох найменших ваг** на кожному кроці дає оптимальне дерево. Інтуїція: щоб мінімізувати сумарну вартість, найкоротші кабелі мають бути найглибше (бо вони увійдуть у суму найбільше разів), а найдовші — найвище (увійдуть найменше разів).

## Приклад

`[4, 3, 2, 6]`:

| крок | heap до       | беремо | вартість | total |
|------|---------------|--------|----------|-------|
| 1    | [2, 3, 4, 6]  | 2 + 3  | 5        | 5     |
| 2    | [4, 5, 6]     | 4 + 5  | 9        | 14    |
| 3    | [6, 9]        | 6 + 9  | 15       | 29    |

**Результат: 29.**

Для порівняння — «погана» стратегія (брати два найдовших спочатку): `6+4=10` (t=10), `3+2=5` (t=15), `5+10=15` (t=30). 30 > 29.
